# 2026-04-25 工作日志

## 修复 Keras 3.x 旧模型加载兼容问题

**问题**：POST /api/prediction/run 报 500，错误为
`ValueError: Could not deserialize 'keras.metrics.mse'` 以及
`ValueError: Unrecognized keyword arguments passed to LSTM: {'time_major': False}`

**根本原因**：旧版 Keras（2.x）保存的 `.h5` 模型在 JSON config 中包含 `time_major: false`，Keras 3.x 的 LSTM 层已移除该参数，导致无法从 config 反序列化重建模型结构。

**追加修复**：`compile=False` 在该版本 Keras 里对旧 `.h5` 同样无效（`compile_args_from_training_config` 仍被调用）。
最终方案：**统一直接走手动路径，完全跳过 `load_model()`**。

**解决方案**：修改 `core/model.py` 的 `load_model` 方法：
- 直接调用 `_load_model_weights_compat(filepath)`，不再尝试 `keras.load_model()`
- 在 `_load_model_weights_compat` 中：
  1. 用 `h5py` 读取 `model_config`
  2. 手动重建 Sequential 网络（过滤掉 `time_major`、`implementation` 等不兼容参数）
  3. 用 dummy input 触发 build
  4. 从 h5 的 `model_weights` 组里逐层 `set_weights()`
- 已验证：加载成功，`predict_point_by_point` 输出正常

**影响范围**：`core/model.py`（`load_model` 方法 + 新增 `_load_model_weights_compat` 方法）
